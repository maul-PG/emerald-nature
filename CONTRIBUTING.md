Terima kasih telah mempertimbangkan untuk berkontribusi ke proyek ini. Panduan singkat di bawah akan membantu menjaga kualitas dan konsistensi kode.

1. Membuka issue terlebih dahulu
- Untuk perubahan besar atau fitur baru, buka issue untuk diskusi desain dan ruang lingkup.

2. Branching
- Gunakan branch terpisah untuk setiap fitur/bugfix.
- Penamaan branch:
   - feature/<deskripsi-singkat>
   - fix/<deskripsi-singkat>

3. Commit messages
- Ikuti aturan singkat (Conventional Commits style):
   - feat: tambahkan fitur
   - fix: perbaiki bug
   - docs: perubahan dokumentasi
   - chore: perubahan kecil/maintenance

4. Style & lint
- Ikuti aturan linting dan formatting yang ada di repo (ESLint/Prettier jika dikonfigurasi). Jalankan linter sebelum membuat PR.

5. Testing
- Tambahkan atau perbarui test untuk setiap perubahan fungsional. Jalankan tes lokal sebelum membuat PR.

6. Pull request
- Buat PR terhadap branch `main`.
- Sertakan deskripsi perubahan, alasan, dan langkah testing.
- Linkkan issue terkait jika ada.

7. Review
- Maintainer akan mereview perubahan. Tanggapi komentar review dan perbaiki jika diperlukan.
